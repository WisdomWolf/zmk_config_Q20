## What happened?
### 2026-03-15
The reusable Github action from upstream zmk project is incompatible with the ZitaoTech fork that defines `bbq20` keyboard.  The build was failing with as issue related to missing 'qualifiers' which appears to be a result of a recent bump to the Zephyr version used in zmk main branch.  The solution was to abandon the reusable build action and recreate the build in this repo.  There were still failures relating to `pointing.h` not being found.  This include was not part of keymaps previously and is apparently added automatically by [Keymap Editor](https://nickcoutsos.github.io/keymap-editor/) as a result of including mouse behavior.  This include breaks the build for the BB Q20 board so I added an extra step to the build to remove this line from the keymap before running the build to prevent breakage when using the Keymap Editor to alter the configuration in the future.
